## 🐛 Исправленные Проблемы

### Проблема №1: Время Считалось Неправильно
**Была:** Все записи показывали 00:01 (1 минута) независимо от времени

**Причина:** 
- `Math.max(1, durationMinutes)` - принудительный минимум
- Старые данные с ошибкой в БД
- Недостаточная валидация дат

**Исправлено:**
- Удалён принудительный минимум
- Очищена БД от старых ошибочных данных
- Добавлена полная валидация timestamp'ов
- Добавлено логирование для отладки
- ✅ Теперь работает правильно!

## 🚀 Как Использовать

### Запуск

```bash
cd d:\work\Time Tracker\time-tracker-app

```

Откроется: `http://localhost:3000`

### Основные Функции

1. **Таймер**
   - Введите название задачи
   - Выберите проект
   - Нажмите START
   - Через 30 сек / 1 мин / N сек нажмите STOP

2. **Проекты**
   - Страница "Projects" → Add New Project
   - Укажите имя и цвет
   - Используйте при создании записей

3. **Отчёты**
   - Страница "Reports"
   - Выберите период (День/Неделя/Месяц)
   - Видите сумму времени по проектам
   - Кнопка "Export CSV"

4. **Редактирование**
   - Нажмите на запись → Edit
   - Измените название/время/проект
   - Сохраните

5. **Удаление**
   - На записи/проекте → кнопка Delete
   - Подтвердить

## 📝 Ключевые Файлы

### API Endpoints

```
POST   /api/time-entries           - Создать запись
GET    /api/time-entries           - Получить записи
GET    /api/time-entries/[id]      - Получить одну запись
PATCH  /api/time-entries/[id]      - Обновить запись
DELETE /api/time-entries/[id]      - Удалить запись
POST   /api/time-entries/[id]/stop - Остановить таймер

POST   /api/projects               - Создать проект
GET    /api/projects               - Получить проекты
PATCH  /api/projects/[id]          - Обновить проект
DELETE /api/projects/[id]          - Удалить проект

GET    /api/reports                - Получить отчёт
GET    /api/reports/export         - CSV экспорт
```

### Ключевые Файлы Исправлений

1. `src/lib/client/time.ts` - Форматирование времени, валидация
2. `src/lib/server/repositories/time-entries-repository.ts` - Расчёт duration
3. `src/app/api/time-entries/[id]/route.ts` - Валидация API
4. `data/time-tracker.db` - Очищена от старых ошибочных записей
